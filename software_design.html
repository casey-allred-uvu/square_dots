Software Design

Model
Data
players: [Player]
An array of players in the game each created from the Player class
current player: Int
index of the current player in the players array
colors: [String]
strings to represent the possible colors to represent the players squares
Functions
init: ([String], Int, Int)
The init function will take a array of strings representing the player names and the width and height of the grid then using other functions initialize the players and grid. Essentially a way to set the inital state up.
initPlayers: ([String])
The init function will take a array of strings representing the player names initialize the players.
initGrid: (Int, Int)
The init function will take the width and height of the grid then initialize the grid.
getPlayersInfo: () -> {name: String,color: String, isTurn: Bool, score: Int}
Return the player info in a generic way the view can understand.
setLineMarked: (Int, Int, String)
Set a certain squares side marked.
getMarkedLines: () -> {x: Int, y: Int, side: String}
Get all the marked lines in the grid in a generic object the view will understand
nextPlayer: ()
Move the current player index to the next player
getCompletedSquares: () -> {x: Int, y: Int, color: String}
Get a list of all completed squares and the color they should be
Classes
Line: selected: Bool
Used to keep track of wether a line is selected (concider removing)
Square: top: Line, right: Line, bottom: Line, left: Line, owner: Player
Used the represent the squares in the grid, the main state of the game. The only function this as is to check if its compelted and assign the passed user as the owner.
Player: name: String, color: String, squares: [Square]
A class to represent the information of the user, including the squares the player owns
